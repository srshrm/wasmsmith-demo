fuzzysearch — typo-tolerant search: WebAssembly vs. JavaScript

Type a search — misspellings are fine. It still finds the right words (serch → search, optimizaton → optimization) and puts the best matches first. The same search runs twice — once in WebAssembly, once in plain JavaScript — so you can compare their speed. Both return the same results, and it all runs in your browser.

A fair race. The JavaScript side is ordinary, readable code — not slowed down on purpose. Against hand-tuned JavaScript the gap narrows to about 1.9×
