simhash — content fingerprinting: WebAssembly vs. JavaScript

Each document is boiled down to a 64-bit fingerprint to catch near-duplicates. This needs exact 64-bit whole-number math — something JavaScript can only fake with its slow BigInt type, while WebAssembly has it built in. Both produce identical fingerprints; only the time differs.
